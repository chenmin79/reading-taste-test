# 阅读品味测试 📚

一个全面的、多轮次的阅读偏好访谈框架，用于推断读者的品味画像并生成个性化书籍推荐。

[English](README.md) | 中文

## 功能特点

- **多维度分析**：超越简单的类型标签，深入理解认知风格、难度容忍度、阅读目的等
- **真实书籍校准**：基于实际阅读历史（喜欢、一般、不喜欢、弃读的书籍）进行精准画像
- **三种测试模式**：简版、标准版、深度版，满足不同精度需求
- **结构化输出**：提供清晰的品味画像和分层推荐
- **双语支持**：支持中文和英文

## 工作原理

测试遵循多轮访谈结构：

1. **第一轮**：快速定位 —— 识别主要阅读类型和认知风格
2. **第二轮**：书籍历史校准 —— 从真实阅读案例推断品味
3. **第三轮**：精细偏好映射 —— 区分相近但不同的品味

## 分析维度

框架追踪 8 个关键维度：

- **内容类型**：虚构 / 非虚构 / 混合
- **偏好领域**：文学、历史、社科、哲学、科普、商业、心理学等
- **认知风格**：叙事驱动、框架驱动、知识密集、论证驱动、好奇心驱动
- **难度容忍度**：轻松、中等、偏难、很难但值得
- **阅读目的**：消遣、获取洞见、自我提升、实用、研究
- **格式偏好**：长篇、散文、密集理论、科普、案例研究等
- **作者/来源偏好**：国内、翻译、经典、当代
- **选书方式**：看评分、看推荐、看主题、看作者、凭好奇心

## 使用方法

### 给 AI 助手使用（ChatGPT、Claude 等）

复制 `reading-taste-test.md` 的内容作为系统提示词或指令，然后引导用户完成多轮访谈。

### 给人工访谈者使用

使用框架文档中提供的问卷模板和流程结构。

## 测试模式

### 简版（3-5 分钟）
- 4-6 个定位问题
- 1 个简短的书籍历史提问
- 简洁画像 + 5-8 本推荐

### 标准版（默认）
- 第一轮：6-8 个定位问题
- 第二轮：3 本喜欢的 / 3 本不喜欢或弃读的书
- 第三轮：3-5 个精细对比问题
- 最终画像 + 分层推荐

### 深度版（高精度）
- 第一轮：定位
- 第二轮：详细阅读历史
- 第三轮：结构和语调偏好
- 第四轮：推荐校准（对候选书籍的反应）
- 最终画像附带置信度说明

## 读者画像类型

框架通过细腻的类型识别读者：

- 知识密集型读者
- 框架构建者
- 叙事探索者
- 实用提升型
- 经典与思想型
- 系统采样者
- 高确定性策展人
- 跨领域通才
- 深挖专家
- 好奇心驱动型探索者

## 输出格式

完整分析包括：

1. **一段式阅读画像** —— 4-8 句话总结阅读风格
2. **关键维度拆解** —— 各维度详细分析
3. **可能的雷区** —— 可能不喜欢或产生摩擦的内容
4. **推荐清单** —— 分层书籍推荐（核心匹配、强匹配、桥梁匹配、延伸匹配）
5. **继续测试建议** —— 进一步提升精度的后续问题

## 快速开始

### 示例对话流程

**AI**：我可以用一个多轮的小测试来判断你的阅读口味，不只是按类型分，还会看你偏好的密度、结构、目标和你真实喜欢/弃读过的书。你想要：
1. 简版（3-5分钟）
2. 标准版（更准）
3. 深度版（最准确）

**用户**：标准版

**AI**：好，标准版开始。先做第一轮定位，你可以直接按题号回答：
1. 你最近更想读：小说 / 历史 / 社科 / 哲学 / 科普 / 传记 / 其他
2. 你读书主要为了：消遣 / 获取知识 / 建立框架 / 开阔视野 / 工作实用 / 情感共鸣
3. 你能接受的难度：轻松 / 中等 / 偏难 / 很难但值得
4. 你更喜欢：叙事推进 / 观点分析 / 案例解释 / 体系化梳理
5. 你更常避开什么：空泛 / 说教 / 术语堆砌 / 太浅 / 太学术 / 太啰嗦

### 标准版问卷模板

**第一部分：基础偏好**
1. 你最常读哪些类型？
2. 你读书最主要是为了什么？
3. 你能接受多高的阅读难度？
4. 你更喜欢叙事、观点、案例还是框架？
5. 你最常放弃哪类书？

**第二部分：真实阅读记录**
6. 说 3 本你很喜欢的书。
7. 说 3 本你觉得一般的书。
8. 说 3 本你不喜欢/弃读的书。
9. 分别简单说下原因。

**第三部分：精细偏好**
10. 更喜欢大框架还是细节？
11. 更喜欢经典还是新书？
12. 更喜欢人的故事还是制度/结构？
13. 更喜欢综述还是深挖？
14. 更喜欢清晰直白还是风格浓烈？

## 使用技巧

1. **不要跳过真实书目** —— 用实际读过的书校准最准确
2. **允许混合答案** —— 用户可能有多种阅读模式
3. **区分"尊重"和"喜欢"** —— 一本书被尊重不代表想要更多类似的
4. **区分"太难"和"太无聊"** —— 两者都导致弃读，但推荐策略不同
5. **关注用户实际说了什么** —— 不要只看类型标签

## 读者类型分析

### 高密度/框架型读者
**特征**：
- 喜欢体系化书籍、大框架解释、知识压缩
- 不喜欢注水、重复、浅薄畅销书
- 通常喜欢思想史、哲学、社科、硬核科普

**推荐风格**：
- 优先推荐密度高但清晰的书
- 解释概念收益
- 避免过于跟风的推荐

### 叙事+洞见型读者
**特征**：
- 要有故事，但不要空洞的故事
- 喜欢小说、回忆录、报道或内嵌思想的叙事非虚构
- 不喜欢纯抽象理论和教科书式阐述

**推荐风格**：
- 推荐洞见由叙事能量承载的书
- 优先考虑语调、声音和情感认知收益

### 实用/模型构建型读者
**特征**：
- 喜欢心理学、决策、经济学、工具、框架
- 会问一本书是否有用、可应用、或能改变世界观
- 如果框架够强，能容忍一定重复

**推荐风格**：
- 区分真正有用的书和注水的机场商业书
- 判断用户想要可操作策略还是持久心智模型

### 经典与思想型读者
**特征**：
- 为风格、模糊性和长期价值接受慢节奏
- 喜欢经典、正典、散文、哲学和文学深度
- 可能不喜欢纯粹效率导向的推荐

**推荐风格**：
- 同时包含易入门和较重的正典作品
- 解释一本经典为何值得投入

## 推荐分层说明

- **核心匹配（Core fit）** —— 很可能会喜欢
- **强匹配（Strong fit）** —— 可能会喜欢
- **桥梁匹配（Bridge fit）** —— 如果好奇可以尝试
- **延伸匹配（Stretch fit）** —— 有用但不保证喜欢

## 常见雷区

- 不要从一两本书过度推断
- 不要把高评分当作唯一信号
- 不要假设文学品味等于非虚构品味
- 不要把"读得多"等同于"偏好明确"
- 如果用户明显有多种阅读模式，不要强行贴单一标签
- 如果用户明确要求精准，不要问太少问题
